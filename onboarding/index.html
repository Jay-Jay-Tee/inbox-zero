Step 1

Connect Gmail (OAuth)

Not verified

This enables dashboard stats + one‑click actions. It uses your Google account permission prompt.

Show me the setup steps
  1. Create a project in Google Cloud Console.
  2. Enable Gmail API for that project.
  3. Set up the OAuth consent screen:
    • Choose External user type.
    • Set publishing status to Testing (for development).
    • Under Test users, add your own Gmail address.
  4. Create an OAuth client ID with application type Chrome Extension.
  5. For Item ID, paste your extension ID from chrome://extensions (Developer mode must be ON to see it).
  6. Copy the OAuth Client ID and paste it into manifest.json under oauth2.client_id.
  7. Go to chrome://extensions and click Reload for this extension.
  8. Come back here and click Test Gmail connection.

Common errors:
403 access_denied → you forgot to add your email under Test users.
bad client id → the OAuth client was created for a different extension ID (Item ID mismatch).

Step 2

Add your Gemini API key

Not verified

This is required for AI summaries and classification. The key is stored in Chrome sync storage.

Show me the steps
  1. Open Google AI Studio – API Keys .
  2. Sign in with the Google account you want to use.
  3. Click Create API key and copy it.
  4. Paste it below and click Save.
  5. Click Verify key works to complete setup.

Common errors:
NO_API_KEY → the key wasn’t saved yet (click Save).
API error 4xx/5xx → the key is invalid, revoked, or rate-limited. Create a new one.
